proper.js / pjs

property-based testing for your javascript

Getting Started

Given that there is no built in string reverse function, lets write one.

Because the results of reversing a reversed string is the original string, we can assert that exact property for reversing all strings.

A list of supported generators is in the README. Have fun!
  1. $ cat string.js 
    String.prototype.reverse = function() {
    return this.split("").reverse().join("");
};
String.props = {
    reverse: function() {
        return FORALL(string(),
            function(s) {
                assert.equal(s, s.reverse().reverse());
            }
        );
    }
};

    $ pjs string.js String
    property String.props.reverse()
    ..................................................
    ..................................................
    OK: Passed 100 test(s).

Install

via Homebrew compiled to use PropEr
  1. $ brew install \
    https://raw.github.com/mokele/homebrew/pjs/Library/Formula/pjs.rb

via Homebrew compiled to use QuviQ QuickCheck. Requires that eqc is in erlang's path already.
  1. $ brew install --with-eqc \
    https://raw.github.com/mokele/homebrew/pjs/Library/Formula/pjs.rb

Through a oneliner
  1. $ curl https://raw.github.com/mokele/proper.js/go | sh

Through a oneliner with QuviQ QuickCheck support.
  1. $ curl https://raw.github.com/mokele/proper.js/go-eqc | sh

Manually
  1. $ git clone https://github.com/mokele/proper.js
  2. $ cd proper.js
  3. $ make
  4. $ PREFIX=/usr/local/pjs make install
  5. $ ln -s /usr/local/pjs/bin/pjs /usr/local/bin/pjs

Alternative make line for QuviQ QuickCheck support.
  1. $ make all cleanebin compile_eqc
